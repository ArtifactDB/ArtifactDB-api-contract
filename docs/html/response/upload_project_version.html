Upload project version response

Type: object

JSON body of the response for the POST /projects/{project}/version/{version}/upload endpoint.

root abort_url
Type: string

Path component for the abort endpoint, which should be appended onto the REST API URL. A PUT request should be performed on the resulting URL, without any body but with all required authorization headers. A successful request should only be possible before upload completion, and will instruct the backend to delete all resources associated with this upload.

root completion_url
Type: string

Path component for the completion endpoint, which should be appended onto the REST API URL. A PUT request should be performed on the resulting URL with all required authorization headers - see the documentation for the PUT /projects{project}/version/{version}/complete endpoint for required parameters in the body. A successful request will indicate to the backend that the upload of this project's version is complete, allowing downstream indexing tasks to start.

Type: string

Path component containing the parameterized link, which should be appended onto the REST API URL. A PUT request should be performed on the full URL; no body is required, but all necessary authorization headers should be added.

A successful request will instruct the backend to create a link from the file specified in the property name to another resource in the ArtifactDB store. The response of the PUT may be ignored.

Type: object

Property names are the relative paths of the files inside the project, while the values are the presigned URLs corresponding to those files.
